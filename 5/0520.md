그동안 공부했던 것들을 다시 살펴보았다. 한번 훑어보는 정도로 봤지만 충격 받은 사실이 하나 있다. class를 알고 있다고 생각했는데 오늘 다시 공부해보니까 내가 알고 있는 것과 달랐다. 그동안 내가 모르고 있었던 것이다. 그래도 오늘에서야 알아서 너무 다행이고 너무 깜짝놀라서 죽을때까지 잊지 못할 것같다. 
여기서 짧게 말하자면
class는 어떤 객체를 생성하기위한 틀
즉, 객체를 새로 만들 때 어떤 객체인지 정의하기 위해서 상태값, 함수 등을 넣어서 구성해두는 곳이다.
```javascript
class Cat {
	constructor(name) {
    	this.name = name
    }
    
   	showName() {
    	console.log(this.name)
    }
}
```
여기서 this는 나 자신을 가리키는 키워드이며 this는 곧 Cat이다.

```javascript
const a = new Cat("pel") // 객체 생성
```
-> 비슷한 여러객체를 만들 때 class를 사용하면 편하다.

```javascript
class MyCat extends Cat {
	constructor(name, age) {
    	super(name) // 부모 클래스에서 가져옴
      	this.age = age;
    }
    showAge() {
     	console.log(this.age) 
    } 
   	showName() {
    	console.log(this.name + "입니다.")
    }
}

const b = new MyCat("pel", 4)
```
-> showName()이라는 메소드가 부모에도 있고 자식에도 있는데 console.log로 찍어보면 "pel입니다."라고 나온다. 즉, 부모클래스에도 있는 함수지만 자식 클래스에 있는 함수가 덮어 씌워서 나온다.
