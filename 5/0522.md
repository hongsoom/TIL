## 알고리즘
알고리즘 주차가 끝났다. 한번도 해본적이 없는 거라서 처음 시작할 때 막막하고 어디부터 시작해야하는지 몰랐는데 알고리즘을 하는 이유부터 어떻게 공부해야하는지, 팀원들과 얘기를 나눠가면서 감을 잡아갔다.
먼저, 무작정 문제를 푸는 것보다 문제 인식하는게 제일 중요하다. 이 문제가 요구하는 것부터 알고 어떤 방식으로 풀어야하는지가 두번째다.
이렇게 한문제씩 풀다보니까 슬슬 문제푸는 속도가 빨라지고 내장함수들을 어떻게 응용하는지, 이 문제에선 어떤 걸 써야하는지 조금씩 알기 시작했다.
스스로 공부할 시간이 많아서 좋았고 많이 사용하는 자바스크립트의 내장함수들도 이제 술술 쓸 수 있고 반복문, 조건문도 파악완료했다. 무엇보다 정말 좋은 팀원들을 만나서 나한테는 아주 좋은 시간이였던것같다. 


## React
리액트를 시작했다. 그래도 한번 공부한 경험이 있어서 낯설지는 않았지만 그때도 그랬고 이번에도 그랬고 역시나 어렵다. 데이터를 주고받는게 정말 헷갈리고 이해가 안간다. 역시나 막막하다. 하지만 똑같이 하다보면 이제 감이 잡힐 것이다. 강의는 그냥 깊게는 안보고 슬쩍 봤지만 다시 한번 봐야할 것 같고 지금은 그냥 아무것도 모르겠다. 다음주도 화이팅하자 열심히 생각하고 고민하다보면 어느순간 이해가 될 것이다.

**JavaScript의 ES란?, ES5/ES6 문법 차이**
ES라는 것은 'Javascript의 표준 규격'을 의미한다.
**ES5**
1. 배열과 관련해서 새로운 메소드들이 생겼는데 대표적으로 forEach, map, filter, reduce, some, every와 같은 메소드가 생겼다. 이 메소드들은 개발자가 반복 횟수나 조건을 잘못 입력하는 등의 실수를 줄여주는 효과가 있다.
2. object에 대한 getter/setter 지원
3. 자바스크립트 strict 모드 지원(더욱 세심하게 문법 검사를 한다.)
4. JSON 지원(과거에는 XML을 사용하다가, json이 뜨면서 지원하게 되었다.)
5. bind() 메소드가 생겼습니다. (this를 강제로 bind 시켜주는 메소드다.)

**ES6**
1. let, const 키워드 추가
2. arrow 문법 지원
3. iterator/generator 추가
4. module import / export 추가
5. Promise 도입
